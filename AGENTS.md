# Working Rules

## Scope

- This is the user's primary URP project.
- Read `PROJECT_MEMORY.md` before meaningful work; update it after durable decisions or completed milestones.

## Safety

- Preserve existing user changes.
- Never commit secrets, `.env`, keys, or generated credentials.
- Ask before destructive, external, or scope-expanding actions.

## Quality

- Keep changes focused and documented.
- Run relevant formatting, linting, and tests after changes once commands exist.
- Update `README.md` when setup or run commands change.
